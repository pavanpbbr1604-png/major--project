# Contribution Guidelines

This document details the standard Git workflow for adding improvements to this repository.

## Future Git Workflows

Whenever code is modified:

1. Stage all modifications:
   ```bash
   git add .
   ```

2. Commit modifications using descriptive and meaningful messages:
   ```bash
   git commit -m "meaningful message"
   ```

3. Push commits to GitHub:
   ```bash
   git push origin main
   ```

## Automated Agent Guidelines

For future AI assistant tasks:
1. Always state which files were modified.
2. Provide a recommended commit message.
3. Do not commit automatically unless explicitly asked.
4. Always preserve Git history.
